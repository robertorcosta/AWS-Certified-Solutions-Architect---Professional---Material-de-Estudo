🌐 Conectividade de Rede ▶
Transit Gateway+
  • Hub central para conectar múltiplas VPCs e redes on-premises
  • Suporta peering inter-Region (tráfego criptografado)
  • Route Tables para segmentação de tráfego
  • Até 50 Gbps por attachment, suporta multicast
  • Transit VIF do Direct Connect conecta ao TGW
💡 50+ VPCs com roteamento centralizado = Transit Gateway
📖 Docs
Direct Connect+
  • Dedicated: 1/10/100 Gbps | Hosted: 50Mbps-10Gbps
  • Private VIF: VPCs | Public VIF: serviços públicos | Transit VIF: TGW
  • LAG para agregar conexões, MACsec para criptografia L2
  • Resiliência: 2 conexões em locais diferentes + VPN fallback
💡 Latência consistente + grande volume de dados = Direct Connect
📖 Docs
Site-to-Site VPN+
  • IPsec via internet, ~1.25 Gbps por túnel (2 túneis/conexão)
  • Accelerated VPN: usa Global Accelerator para melhor performance
  • ECMP com Transit Gateway: agregar múltiplas VPNs para mais throughput
💡 Backup do DX ou conexão rápida temporária = VPN
📖 Docs
VPC Peering+
  • 1-para-1, cross-region, cross-account
  • NÃO transitivo — não pode rotear por peering intermediário
  • Sem custo de hora, paga só transferência de dados
  • Pode referenciar Security Groups de VPC peerada (cross-account!)
📖 Docs
PrivateLink+
  • Expõe um serviço para outras VPCs/contas sem expor na internet
  • Interface Endpoint (ENI + IP privado) na subnet do consumidor
  • Gateway Endpoint (S3 e DynamoDB): grátis, rota na route table
💡 "Acessar S3 sem NAT Gateway" = VPC Gateway Endpoint (grátis!)
📖 Docs
Route 53 Resolver+
  • Inbound Endpoint: on-prem resolve DNS na AWS
  • Outbound Endpoint: AWS resolve DNS on-prem
  • Resolver Rules compartilháveis via RAM (multi-account)
📖 Docs
🔒 Segurança & IAM ▶
IAM Cross-Account+
  • AssumeRole: método preferido (temporário, auditável)
  • Resource-based Policy: sem perder permissões originais
  • SCPs: guardrails no nível de Organization
  • Permission Boundaries: teto máximo para roles/users
💡 Deny explícito SEMPRE vence. Cross-account: ambos lados precisam Allow.
📖 Docs
IAM Identity Center+
  • SSO para múltiplas contas + apps SAML 2.0 + EC2 Windows
  • Permission Sets atribuídos a users/groups por conta
  • ABAC: permissões baseadas em atributos (cost center, title)
  • Identity sources: built-in, AD, Okta, Azure AD
📖 Docs
KMS+
  • Symmetric (AES-256): encrypt/decrypt, envelope encryption
  • Asymmetric (RSA/ECC): sign/verify, encrypt fora da AWS
  • Multi-Region Keys: encrypt em uma região, decrypt em outra
  • Custom Key Store: material no CloudHSM
  • Keys são REGIONAIS (exceto multi-region)
📖 Docs
CloudTrail+
  • Auditoria de API calls (management events por padrão)
  • Data Events (S3/Lambda): opt-in, alto volume
  • Insights: detecta atividade incomum com ML
  • Retenção: 90 dias → S3 + Athena para long-term
  • Organization Trail: todas as contas, um trail
💡 Reagir rápido a eventos: EventBridge (mais rápido) > CW Logs > S3
📖 Docs
GuardDuty / Security Hub+
  • GuardDuty: detecção de ameaças com ML (findings)
  • Security Hub: visão consolidada + compliance (CIS, PCI)
  • Inspector: vulnerabilidades em EC2/ECR/Lambda
  • Macie: dados sensíveis em S3 (PII)
  • Todos suportam delegated admin para conta de segurança

🔄 Resiliência & DR ▶
Estratégias de DR+
  • Backup & Restore: horas RTO, baixo custo
  • Pilot Light: core mínimo rodando, escala no DR
  • Warm Standby: sistema reduzido ativo, minutos RTO
  • Multi-Site: full active-active, near-zero RTO
💡 "Menor custo" + DR = Backup & Restore. "RTO mínimo" = Multi-Site.
📖 Docs
Aurora Global / DynamoDB Global+
  • Aurora Global: RPO < 1s, RTO < 1min, até 5 regiões
  • DynamoDB Global Tables: multi-region active-active, auto-replication
  • ElastiCache Global Datastore: Redis cross-region
📖 Docs
Elastic Disaster Recovery+
  • Replicação contínua block-level de servidores para DR region
  • Failover em minutos, suporta failback
  • Substitui CloudEndure Disaster Recovery
📖 Docs
Route 53 Routing Policies+
  • Failover: ativo/passivo com health checks
  • Latency-based: menor latência para o user
  • Weighted: % do tráfego (blue/green, canary)
  • Geolocation/Geoproximity: por localização
  • Multivalue Answer: múltiplos IPs saudáveis
📖 Docs
AWS Backup / Vault Lock+
  • Centraliza backups cross-service (EC2, RDS, S3, EFS, DynamoDB...)
  • Cross-region e cross-account, PITR, tag-based policies
  • Vault Lock (WORM): nem root user pode deletar backups!
  • Proteção contra ransomware e delete acidental
📖 Docs
Fault Injection Simulator+

⚡ Performance & Cache ▶
CloudFront+
  • CDN global (HTTP/HTTPS), cache de static + dynamic
  • Lambda@Edge / CloudFront Functions para lógica no edge
  • Origin failover para HA
  • Custo egress menor que direto do S3/ALB
📖 Docs
ElastiCache (Redis/Memcached)+
  • Redis: Multi-AZ, replicação, persistência, rich data types, Global Datastore
  • Memcached: simples, multi-thread, sem HA/replicação
💡 Precisa de HA/persistência = Redis. Cache simples horizontal = Memcached.
📖 Docs
Global Accelerator+
  • Anycast IPs (2 estáticos), camada 4 (TCP/UDP)
  • Failover instantâneo entre regiões via health checks
  • Ideal para: gaming, IoT, VoIP, apps que precisam de IP fixo
💡 vs CloudFront: GA = sem cache, TCP/UDP, IP fixo. CF = cache HTTP, DNS.

Criptografia em Trânsito+
  • ACM: certificados TLS gerenciados (grátis para serviços AWS)
  • VPN: IPsec para Site-to-Site
  • Client VPN: OpenVPN para acesso remoto de users
  • Tudo entre AWS e internet = TLS por padrão
📖 Docs
Criptografia em Repouso+
  • S3: SSE-S3 (padrão) | SSE-KMS (auditável) | SSE-C (sua chave)
  • EBS: KMS, habilitável por região (account-level setting)
  • RDS: criptografa na criação (snapshot → copy encrypted → restore)
  • DynamoDB: AWS owned key ou CMK
💡 RDS não criptografado → snapshot → copy com encryption → restore
Secrets Manager vs Parameter Store+
  • Secrets Manager: $0.40/mês, rotação automática (Lambda), KMS obrigatório, cross-account via resource policy
  • Parameter Store: grátis (standard), hierarquia (/app/prod/key), sem rotação nativa, 4KB max
💡 "Rotação automática de DB creds" = Secrets Manager. "Config com hierarquia" = Parameter Store.

💰 Otimizar Custos ▶
Identificar Desperdícios+
  • EBS volumes não anexados (você paga!)
  • Elastic IPs não associados (cobra quando NÃO está em uso)
  • Instâncias oversized (CPU < 10%)
  • NAT Gateway com muito tráfego → VPC Endpoints
  • S3 sem lifecycle policies → dados na classe errada
  • Snapshots/AMIs antigos acumulando custo
Automação de Custos+
  • Instance Scheduler: liga/desliga EC2/RDS em horário comercial
  • Budgets Actions: aplica SCP ou para instâncias ao exceder budget
  • Tagging obrigatório: SCP bloqueia criação sem tags de custo

Migração & Modernização Domínio 4 — 20%
✈️ Estratégias de Migração ▶
7Rs da Migração+
  • Retire: desligar o que não precisa
  • Retain: manter onde está (por enquanto)
  • Rehost: lift & shift (MGN)
  • Relocate: VMware → VMware on AWS
  • Replatform: lift & reshape (MySQL → RDS)
  • Repurchase: trocar por SaaS
  • Refactor: redesenhar cloud-native
💡 "Menor esforço" = Rehost. "Melhor uso da cloud" = Refactor.
MGN (App Migration Service)+
  • Rehost em larga escala (substitui CloudEndure + SMS)
  • Replicação contínua → test launch → cutover
  • Suporta: VMs, físicos, qualquer cloud → AWS
📖 Docs
DMS + SCT+
  • DMS: Full Load + CDC para mínimo downtime
  • SCT: converte schema (só para heterogêneo)
  • Snowball + DMS: SCT extrai → Snow → S3 → DMS migra
  • Suporta Oracle, SQL Server, MySQL, PostgreSQL, MongoDB → AWS
💡 Oracle → Aurora: SCT (schema) + DMS (dados com CDC)
📖 Docs
